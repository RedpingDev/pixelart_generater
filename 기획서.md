# 픽셀 아트 생성 서비스 기획서 (개인용 MVP)

> **목표**: 혼자 사용할 픽셀 아트 생성 도구를 최소 비용·최단 시간으로 구축
> **방식**: 경로 A (외부 API 활용형)
> **작성일**: 2026-04-21

---

## 1. 프로젝트 개요

### 1.1 목적
- 게임 에셋·취미용 픽셀 아트를 프롬프트만으로 생성
- AetherForge 같은 상용 서비스의 핵심 기능을 개인용으로 압축
- 운영비 월 1만원 이하, GPU 자체 운영 없음

### 1.2 사용자
- **본인 1명** (인증 시스템 단순화 가능, 또는 생략)

### 1.3 핵심 가치
- "한 줄 입력 → 30초 안에 픽셀 아트 결과물"
- 결과물 히스토리 저장 및 재사용
- 부분 수정(Inpainting) 지원

---

## 2. 핵심 기능 (MVP 범위)

| 우선순위 | 기능 | 설명 |
|---|---|---|
| P0 | 텍스트 → 픽셀 아트 생성 | 프롬프트 입력 → 이미지 1~4장 생성 |
| P0 | 결과 다운로드 | PNG (배경 투명 옵션) |
| P0 | 히스토리 보기 | 과거 생성물 갤러리 |
| P1 | 프롬프트 자동 보강 | GPT가 픽셀 아트용 프롬프트로 확장 |
| P1 | 배경 제거 | rembg API로 투명 배경 처리 |
| P1 | 픽셀 사이즈 선택 | 32x32 / 64x64 / 128x128 |
| P2 | 부분 수정 (Inpaint) | 마스크 그려서 일부만 재생성 |
| P2 | 스프라이트 시트 | 동일 캐릭터 여러 포즈 |
| P3 | 팔레트 강제 지정 | 16/32색 양자화 |

## 3. 기술 스택

### 3.1 확정 스택
```
Frontend : Next.js 14 (App Router) + TypeScript
스타일   : Tailwind CSS + shadcn/ui
백엔드   : Next.js API Routes (서버리스)
DB       : Supabase (PostgreSQL + Storage)
AI 생성  : Replicate API
            - 모델: nerijs/pixel-art-xl (SDXL + Pixel LoRA)
AI 보강  : OpenAI API (gpt-4o-mini, 프롬프트 확장용)
배경 제거: Replicate (cjwbw/rembg)
배포     : Vercel (무료 티어)
이미지 처리: sharp (Node) — 다운스케일·팔레트 양자화
```

### 3.2 선택 기준
- **Replicate**: 호출당 과금($0.003~0.01), GPU 운영 불필요
- **Supabase**: 무료 500MB DB + 1GB Storage, 인증 무료 제공
- **Vercel**: Next.js 무료 호스팅
- **혼자 사용**: 동시 처리·스케일링 고민 불필요

## 4. 시스템 아키텍처

```
┌─────────────┐
│  브라우저   │  Next.js UI (프롬프트 입력 / 갤러리)
└──────┬──────┘
       │ HTTPS
       ▼
┌─────────────────────────────┐
│  Vercel (Next.js Server)   │
│  ┌───────────────────────┐ │
│  │ /api/generate         │ │ ──► OpenAI (프롬프트 보강)
│  │ /api/inpaint          │ │ ──► Replicate (이미지 생성)
│  │ /api/remove-bg        │ │ ──► Replicate (배경 제거)
│  └───────────────────────┘ │
└──────────────┬──────────────┘
               ▼
       ┌───────────────┐
       │   Supabase    │
       │  - 메타데이터 │
       │  - 이미지 저장│
       └───────────────┘
```

### 처리 플로우 (생성)
```
1. 사용자: "용을 타는 마법사" 입력
2. /api/generate 호출
3. (옵션) GPT가 → "A wizard riding a red dragon, pixel art,
   16-bit, vibrant colors, white background" 으로 보강
4. Replicate pixel-art-xl 호출 (5~15초)
5. 결과 이미지 → sharp로 다운스케일·업스케일 (선명한 픽셀화)
6. Supabase Storage 업로드
7. DB에 메타데이터 저장 (프롬프트, URL, 시각)
8. 클라이언트로 URL 반환 → 화면 표시
```

## 5. 데이터 모델

### Supabase 테이블

```sql
-- 생성 이력
create table generations (
  id           uuid primary key default gen_random_uuid(),
  prompt_raw   text not null,
  prompt_final text not null,
  image_url    text not null,
  pixel_size   int  default 64,
  bg_removed   boolean default false,
  model        text default 'pixel-art-xl',
  cost_usd     numeric(10,4),
  created_at   timestamptz default now()
);

-- 즐겨찾기
create table favorites (
  generation_id uuid references generations(id) on delete cascade,
  created_at    timestamptz default now(),
  primary key (generation_id)
);
```

### Storage 버킷
- `pixel-art/` (public read) — 생성된 PNG 저장

## 7. API 명세 (3개)

### POST `/api/generate`
```json
요청: {
  "prompt": "용을 타는 마법사",
  "size": 64,
  "removeBg": true,
  "enhance": true,
  "count": 4
}
응답: {
  "items": [
    { "id": "uuid", "url": "https://...png", "promptFinal": "..." }
  ]
}
```

### POST `/api/remove-bg`
```json
{ "imageUrl": "..." } → { "url": "..." }
```

### POST `/api/inpaint` (P2)
```json
{ "imageUrl": "...", "maskUrl": "...", "prompt": "..." } → { "url": "..." }
```

## 9. 보안·운영

- **API 키 보호**: `.env.local` + Vercel 환경변수, 클라이언트 노출 금지
- **인증** (혼자 사용): Supabase Auth 매직 링크 1계정만 허용, 또는 미들웨어에서 `BASIC_AUTH_PASSWORD` 검증
- **Rate Limit**: Vercel Edge Middleware로 IP당 분당 10회
- **백업**: Supabase Storage 주기적 로컬 다운로드 (선택)
